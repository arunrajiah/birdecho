# Good-neighbour checklist

Things to do before and after posting or opening PRs in upstream communities.

## Before posting anywhere

- [ ] Read the community's CONTRIBUTING guide and CODE_OF_CONDUCT
- [ ] Check recent posts/issues to make sure no one else has already posted about BirdEcho
- [ ] Make sure the GitHub release is live and the APK link works
- [ ] Make sure the README accurately reflects current features

## BirdNET-Pi (mcguirepr89/BirdNET-Pi)

- [ ] Read the Discussions tab — is there a "community tools" or "related projects" thread?
- [ ] If yes, reply to that thread instead of opening a PR
- [ ] If a PR is appropriate, follow the PR template in the repo
- [ ] Wait at least 1 week before following up on an unanswered PR

## BirdNET-Go (tphakala/birdnet-go)

- [ ] Same checklist as BirdNET-Pi above
- [ ] Note: tphakala is responsive on GitHub — a Discussions post may get faster feedback than a PR

## BirdWeather

- [ ] Use the contact form rather than a public GitHub issue
- [ ] Give them at least 2 weeks to respond before mentioning the app publicly in their community channels
- [ ] Do not post in their community channels as "promoted by BirdWeather" unless they explicitly say so

## General rules

- Never describe BirdEcho as "official", "recommended by", or "endorsed by" any upstream project
- Always include the GitHub link so people can inspect the source
- Respond to every question in a thread, even if it's critical
- If someone reports a bug in a community thread, direct them to GitHub Issues rather than debugging in comments
- Thank maintainers for their work when interacting with their repos
